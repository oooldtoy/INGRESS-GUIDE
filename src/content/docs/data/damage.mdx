Ingress 中的伤害计算相对简单，但拥有几项独特的机制。整体而言，伤害主要受**距离衰减**、**暴击**和 **[防御值](/INGRESS-GUIDE/reference/mitigation/)** 的影响。

### 1. 阶梯式距离衰减

武器（如 XMP Burster、Ultra Strike）的伤害随距离衰减，衰减并非线性，而是根据距离目标（Resonator 或 Portal 中心）的比例呈**阶梯式下降**。

将武器的**攻击生效范围**分为五个区间。假设 <i>r</i> 为（玩家到目标的实际距离 ÷ 武器最大攻击距离）：

<ul>
  <li><b>距离最近</b>（0 ≤ <i>r</i> ≤ 20%）：造成 <b>100%</b> 伤害</li>
  <li><b>距离次近</b>（20% &lt; <i>r</i> ≤ 40%）：造成 <b>50%</b> 伤害</li>
  <li><b>距离中等</b>（40% &lt; <i>r</i> ≤ 60%）：造成 <b>25%</b> 伤害</li>
  <li><b>距离较远</b>（60% &lt; <i>r</i> ≤ 80%）：造成 <b>12.5%</b> 伤害</li>
  <li><b>距离最远</b>（80% &lt; <i>r</i> ≤ 100%）：造成 <b>6.25%</b> 伤害</li>
</ul>

### 2. 攻击目标与结算方式

攻击范围内的可见实体包含 **Resonator** 和 **Portal 本体（即 Mod）**，两者的受击结算是独立分开的：

#### 对 Resonator的攻击
- 伤害基于玩家位置到**每一个独立 Resonator** 的实际距离进行上述阶梯衰减计算。
- 扣除对应 Resonator 的 XM（能量）。

#### 对 Portal 本体的攻击
- 伤害基于玩家到 **Portal 中心**的距离进行计算。
- **Portal 本体不会掉血**。对 Portal 的攻击主要用于破坏其部署的 **Mod**，而这一过程完全取决于**是否触发暴击**。

### 3. 暴击与 Mod 破坏

攻击有一定概率触发暴击，暴击率受到武器等级和距离的影响，[Ultra Strike](/INGRESS-GUIDE/reference/us/)拥有比[XMP](/INGRESS-GUIDE/reference/xmp/)更高的暴击率，更适合用来破坏Mod。

虽然没有直接证据，但是据信暴击率与攻击力同样遵循阶梯式衰减，即每过1/5最大攻击距离，暴击率减半。
- 如果对 Resonator 触发暴击，会对其造成 **2 倍**的基础伤害。
- Mod 脱落判定：暴击是对 Portal 上的 Mod 触发破坏判定的**唯一条件**。
  - 每次暴击时，每个 Mod 会**单独**进行一次脱落判定。
  - Mod 的 **[黏性](/INGRESS-GUIDE/reference/stickiness/)** 决定了它的抵抗力：例如，一个具有 40% 黏性的 Mod，在遭到暴击时有 40% 的概率不脱落（即 60% 的概率被破坏）。
  - 如果一个 Mod 的黏性为 0%，则只要遭遇暴击必定脱落。

### 4. 防御减免

Resonator 受到的最终伤害会受到 Portal **[防御值](/INGRESS-GUIDE/reference/mitigation/)** 的百分比减免。

- 防御值直接等于减伤百分比。例如，Portal 拥有 30 防御值，即代表最终减伤 30%。
- Mod 和 Link 提供的最大有效减免上限为 **95%**。超出 95% 的防御值会作为“储备防御”生效。
  - 例如：3 个 Aegis Shield（每个 70 防御值）总共可提供 210 防御值，此时实际生效的最终减伤为上限的 **95%**。
  - 如果其中 1 个 Aegis Shield 被打掉，剩余 140 防御值，实际表现出的依然是 **95%** 减伤。
  - 如果继续被打掉 1 个，只剩最后 1 个 Aegis Shield，防御值降为 70，此时实际表现就变成了 **70%** 减伤。

### 5. 最终伤害计算公式

综合以上机制，对 Resonator 的最终伤害判定公式可以表示为：

<div style={{
  display: 'flex',
  justifyContent: 'center',
  alignItems: 'center',
  fontFamily: '"Cambria Math", "Times New Roman", serif',
  fontSize: '1.25rem',
  margin: '2rem 0',
  flexWrap: 'wrap',
  gap: '12px'
}}>
  <i>D<sub>Final</sub></i> = <i>D<sub>Base</sub></i> &times; <i>E(r)</i> &times; <i>C<sub>crit</sub></i> &times;(1+<i>C<sub>Bonus</sub>%</i>) &times;    [ 1 &minus; <i>min</i>( 95%, <i>M</i> ) ]
</div>

其中：
- <i>D<sub>Base</sub></i> 为武器基础伤害
- <i>E(r)</i> 为距离衰减系数 (根据距离分别为1.0, 0.5, 0.25, 0.125, 0.0625)
- <i>C<sub>crit</sub></i> 为暴击系数 (触发=2, 未触发=1)
- <i>C<sub>Bonus</sub></i> 为[蓄力攻击奖励](/INGRESS-GUIDE/guides/attack/#蓄力攻击)
- <i>M</i> 为 Portal 的总防御减免百分比

> **注意：** 该公式仅计算对 Resonator 造成的实质性 XM（血量）伤害。对于 Portal 本体的 Mod，仅在暴击时根据 Mod 的黏性概率计算是否破坏，不涉及血量计算。

### 6. 蓄力下DPS对比

考虑激烈攻防情况下的每秒伤害量（Damage Per Second, DPS），对比两种极端情况：

1. 永远完美蓄力
2. 完全不蓄力

以下计算均针对L8 XMP/US做出。作出以下前提假设：

1. 攻击时道具无法无缝连续释放，在松开 Attack 按钮后有一段时间该按钮会灰掉。这个时间<i>t<sub>i</sub></i>与系统固定延迟及网络延迟有关，根据观察，<i>t<sub>i</sub> > 0.8s</i>。
2. 对于L8攻击道具，20%完美蓄力的时间<i>t<sub>c</sub></i>约为0.56s，本地计算不受网络延迟影响。
3. 蓄力对暴击与攻击力均有加成。

代入 <i>t<sub>c</sub></i> ≈ 0.56s 得到临界值约为 **1.31s**，结论如下：

- **<i>t<sub>i</sub></i> &lt; 1.31s**（延迟较低）：不蓄力连点 DPS 更高。在 <i>t<sub>i</sub></i> 接近下限 0.8s 的极低延迟情况下，不蓄力比蓄力高出约 **19%** 的 DPS。
- **<i>t<sub>i</sub></i> > 1.31s**（延迟较高）：蓄力攻击 DPS 反超，随延迟增大优势逐渐扩大。

也就是说，蓄力是否合算完全取决于实际网络状况。然而，玩家很难做到连续完美蓄力，网络延迟也不会达到如此之高，因此不蓄力的DPS常常更高。

即使如此，蓄力也是 **最经济** 的进攻方式，毕竟单发XMP的期望伤害更高（提高了42.8%）。在竞争不激烈或物资吃紧的情况下，蓄力仍然是一个很优秀的选择。
