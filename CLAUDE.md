This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview
- This is an **Astro 6 + Starlight** documentation site (not an app backend/service).
- Docs content is sourced from `src/content/docs/` via Astro Content Collections configured in `src/content.config.ts`.
- Site branding/navigation is centralized in `astro.config.mjs` (title, sidebar autogeneration, GitHub Pages URL/base path).

## Common Commands
Run from repository root.

- Install dependencies:
  - `npm install`
- Start local dev server:
  - `npm run dev`
- Build production site:
  - `npm run build`
- Preview production build locally:
  - `npm run preview`
- Run Astro CLI subcommands:
  - `npm run astro -- <command>`
  - Example project validation/type checks: `npm run astro -- check`

## Linting and Tests
- There is currently **no dedicated lint script** in `package.json`.
- There is currently **no test framework/script configured** in `package.json`.
- Single-test execution is therefore **not applicable yet** in the current setup.

## Architecture and Key Conventions
- **Docs collection wiring**: `src/content.config.ts` defines a single `docs` collection using Starlight’s `docsLoader()` and `docsSchema()`.
- **Routing/content model**: each `.md`/`.mdx` file under `src/content/docs/` becomes a docs route.
- **Navigation model**: sidebar in `astro.config.mjs` is autogenerated from these directories:
  - `guides`
  - `data`
  - `play`
  Keep content organized under these folders so it appears correctly in navigation.
- **Homepage**: `src/content/docs/index.mdx` is the landing page content and hero config; it references `src/assets/ingress.png`.
- **Static assets**:
  - Content images: typically under `src/assets/`
  - Public static files (e.g. favicon): `public/`

## Deployment
- GitHub Actions workflow: `.github/workflows/deploy.yml`.
- Deployment triggers on pushes to `master`.
- Uses `withastro/action@v3` with **Node 22** for Astro 6 compatibility.
- GitHub Pages pathing is tied to `astro.config.mjs`:
  - `site: https://MarsianCreator.github.io/INGRESS-GUIDE/`
  - `base: /INGRESS-GUIDE`
  Preserve these settings unless intentionally changing repository/site URL structure.
